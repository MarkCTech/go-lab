Privileged: support acknowledgment

This is a Phase A control-plane sample: support staff can record that they performed or acknowledged a sensitive support action (policy, incident, customer handling) with a mandatory written reason. The API writes an immutable row to admin_audit_events (support.ack) with your user id, the reason header, optional message, and request metadata. It is not a ticket system; it is an audit trail so later reviews can answer “who acknowledged what, and why.” Same pattern as DataOps restore mutations (reason header + rate limits).

POST /api/v1/support/ack · header X-Platform-Action-Reason (min 10 chars) · permission platform.support.ack (e.g. operator or support role).
